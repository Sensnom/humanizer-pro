# Humanizer Pro

Humanizer Pro 是一个中文文本改写与审阅用的 Claude Skill。它主要处理中文写作里的 AI 味：模板句、宣传腔、翻译腔、空泛总结、机械排比，以及过度完整但不太像人写的表达。

它的目标不是把文字改得更花哨，而是在保留原意、事实和语气的前提下，让表达更自然、更贴近对应场景。

## 适合处理什么

Humanizer Pro 更适合中文文本的自然化修改，尤其是下面几类内容：

- 课程论文、实验报告、结课作业
- 技术说明、工具介绍、项目文档
- 心得体会、总结、申请材料、思想汇报
- 聊天回复、评论、私信
- 公众号文章、博客、观点文
- 百科式说明、资料整理
- 英文翻译后中文不够顺的文本

常见问题包括：

- “具有重要意义”“深刻认识到”“从多个维度”这类空泛表达太多
- 每段都像按“首先、其次、最后”凑出来的
- 技术说明写得像广告或宣传稿
- 聊天回复像客服、心理咨询师，或者像 AI 在安慰人
- 句子过于完整、平衡、正确，但读起来不太像正常写作
- 使用了没有来源的“专家认为”“研究表明”“多个来源显示”
- 过度使用冒号、破折号、引号、粗体和项目符号

## 它会怎么处理文本

主要处理流程是：

1. 判断文本场景，比如课程报告、技术说明、聊天回复或公众号文章。
2. 保留原文里的事实、数据、术语、态度和基本逻辑。
3. 找出真正影响自然度的部分，例如空泛升华、模板句、宣传腔、翻译腔。
4. 按场景改写，而不是一律改成口语化表达。
5. 复查是否改丢信息、误加事实，或把正式文本改得太随意。

## 文件结构

```text
humanizer-pro/
├── SKILL.md    # Claude Skill 主文件
├── README.md   # 项目说明
└── LICENSE     # MIT License
```

`SKILL.md` 是核心文件，里面包含触发描述、工作流程、场景规则、句式规则、词汇规则和示例。Claude Code 会根据 `SKILL.md` 中的 description 判断什么时候使用这个 skill。

## 安装

### 使用 skills CLI 安装

```bash
npx skills add https://github.com/Sensnom/humanizer-pro.git
```

### 手动安装

也可以把仓库克隆到 Claude Code 的 skills 目录：

```bash
git clone git@github.com:Sensnom/humanizer-pro.git ~/.claude/skills/humanizer-pro
```

安装后，在 Claude Code 里提出类似“帮我去 AI 味”“改得自然一点”“审阅这段有没有 AI 感”的请求时，Claude Code 会根据 skill 描述决定是否调用它。

## 使用示例

### 课程报告

```text
帮我把下面这段话去 AI 味，保留原意，不要改得太口语。

本实验不仅帮助学生深入理解理论知识，更培养了学生的实践能力和科学素养，体现了理论与实践相结合的重要意义。
```

可能改成：

```text
本实验把理论公式和实际测量过程对应起来，也暴露出仪器读数、参数设置和误差处理中的具体问题。
```

### 技术介绍

```text
帮我把这段技术介绍改得更像正常项目文档，不要像广告。

该平台凭借强大的生态和灵活的扩展能力，为开发者提供了一站式解决方案。
```

可能改成：

```text
该平台提供模型调用、数据管理和接口调试功能。开发者可以在同一环境中完成测试和部署前检查。
```

### 聊天回复

```text
帮我把这句话改得像朋友聊天，不要像 AI 在安慰人。

我能感受到你现在真的很开心，这种等待很久终于实现目标的感觉一定很珍贵。
```

可能改成：

```text
等了这么久，总算有结果了，肯定很激动吧。
```

更口语一点：

```text
终于等到了，太不容易了。
```

### 审阅 AI 痕迹

```text
帮我看看下面这段话哪里像 AI 写的，先指出问题，再给一版改写。

随着人工智能技术的不断发展，其在教育领域的应用展现出广阔前景。它不仅提升了教学效率，也为学生个性化学习提供了新的可能，具有重要的现实意义。
```

可能指出的问题：

- “随着……不断发展”是常见套话，信息量不高。
- “广阔前景”“新的可能”“重要的现实意义”偏空泛，没有说明具体作用。
- 句子结构比较完整，但缺少具体场景和对象。

可能改成：

```text
人工智能已经开始用于作业批改、学习反馈和课堂辅助。它能减少一部分重复性工作，也方便教师根据学生的答题情况调整讲解重点。
```

## 推荐 prompt

如果你希望结果更稳定，可以说明文本用途和改写边界：

```text
请帮我处理下面这段文字。

用途：课程论文 / 技术说明 / 心得体会 / 聊天回复 / 公众号文章
要求：保留原意，不新增事实，去掉 AI 味，保持自然，但不要过度口语化。
输出：先给改写版本，再简单说明主要改了哪里。

原文：
...
```

如果文本较长，建议分段处理。一次处理太多内容时，模型可能为了整体流畅而改动细节。

## 处理边界

这个 skill 是文本编辑辅助工具，不用于保证“AI 检测通过”。

它不能保证：

- 绕过任何 AI 检测工具
- 自动判断事实真伪
- 自动补全真实经历、数据或引用来源
- 替代引用核查、论文审稿或人工编辑

如果原文里的事实、数据、来源本身不确定，应该先核验，再改写。
