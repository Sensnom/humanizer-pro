# Humanizer Pro

中文去 AI 味的 Claude Skill。

它用于审阅和改写中文文本：找出模板感、宣传腔、翻译腔、过度完整的句式和空泛表达，再在保留原意的前提下改得自然一点。

“自然”不是故意加错别字，也不是把所有句子都改成口语。课程论文还是要像课程论文，技术说明还是要像技术说明，聊天回复才需要像正常聊天。

## 适用场景

- 课程论文、实验报告、结课作业
- 技术说明、工具介绍、项目文档
- 心得体会、总结、申请材料、思想汇报
- 聊天回复、评论、私信
- 公众号文章、博客、观点文
- 百科式说明、资料整理
- 英文翻译腔较重的中文

适合处理的问题包括：

- “具有重要意义”“深刻认识到”“从多个维度”这类空泛表达太多
- 每段都像“首先、其次、最后”凑出来的
- 技术说明写得像广告
- 聊天回复像客服或心理咨询师
- 句子过于完整、平衡、正确，但读起来不像正常人写的
- 用了很多没有来源的“专家认为”“研究表明”“多个来源显示”

## 文件结构

```text
humanizer-pro/
├── SKILL.md
├── README.md
└── LICENSE
```

`SKILL.md` 是 Claude Skill 主文件，包含触发描述、工作流程、场景规则、词汇规则、句式规则和示例。

## 安装

使用 skills CLI 安装：

```bash
npx skills add https://github.com/Sensnom/humanizer-pro.git
```

也可以手动克隆到 Claude Code 的 skills 目录：

```bash
git clone git@github.com:Sensnom/humanizer-pro.git ~/.claude/skills/humanizer-pro
```

安装后，Claude Code 会根据 `SKILL.md` 里的 description 判断什么时候使用它。

## 使用示例

### 课程报告

```text
帮我把下面这段话去 AI 味，保留原意，不要改得太口语。

本实验不仅帮助学生深入理解理论知识，更培养了学生的实践能力和科学素养，体现了理论与实践相结合的重要意义。
```

可能改成：

```text
本实验把理论公式和实际测量过程对应起来，也暴露出仪器读数、参数设置和误差处理中的具体问题。
```

### 技术介绍

```text
帮我把这段技术介绍改得更像正常项目文档，不要像广告。

该平台凭借强大的生态和灵活的扩展能力，为开发者提供了一站式解决方案。
```

可能改成：

```text
该平台提供模型调用、数据管理和接口调试功能。开发者可以在同一环境中完成测试和部署前检查。
```

### 聊天回复

```text
帮我把这句话改得像朋友聊天，不要像 AI 在安慰人。

我能感受到你现在真的很开心，这种等待很久终于实现目标的感觉一定很珍贵。
```

可能改成：

```text
等了这么久，总算有结果了，肯定很激动。
```

更口语一点：

```text
终于等到了，太不容易了。
```

## 推荐 prompt

```text
请帮我处理下面这段文字。
用途：课程论文 / 技术说明 / 心得体会 / 聊天回复 / 公众号文章
要求：保留原意，不新增事实，去掉 AI 味，保持自然但不要过度口语化。
输出：先给改写版本，再简单说明主要改了哪里。

原文：
...
```

长文本建议分段处理。一次处理太多内容，容易为了整体流畅而改动细节。

## 注意事项

这个 skill 是文本编辑辅助工具，不用于保证“AI 检测通过”。

它不能保证：

- 绕过任何 AI 检测工具
- 自动判断事实真伪
- 自动补全真实经历或数据
- 替代引用核查、论文审稿或人工编辑
