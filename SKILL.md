# Humanizer Pro

## 1. 使用目标

本技能用于审阅和改写中文文本。目标是去除 AI 写作痕迹，让文本更自然、流畅、贴近人类写作风格，同时保留原文的事实、逻辑和态度。适用于课程论文、技术说明、心得体会、社交回复、公众号文章等多种文本类型。

改写前先判断文本用途。课程报告、技术说明、聊天回复、心得体会、公众号文章的写法不同，不能套用同一种自然化方式。

处理时遵守以下原则：

1. 保留原意、事实、数据、术语、态度和逻辑顺序。
2. 不添加原文没有的经历、数据、引用、来源或结论。
3. 不为了自然感故意制造错别字、病句、松散句或过度口语化表达。
4. 不把句子一律改短。必要的过渡词、虚词和长句可以保留。
5. 优先用普通、直接、具体的说法，不用抽象包装词替代简单意思。
6. 改写结果要符合文本所在场景，而不是统一变成“聊天感”或“说明书感”。
7. 减少 meta 表达。不要在正文里写“下面是改写版本”“我主要做了这些调整”“这段文字的修改思路是”等过程说明，除非用户明确要求解释。

本技能自身输出也要遵守以上原则。解释修改原因时，少用包装性引号、抽象概念和过度完整的自我说明。反例、术语名、文件名、章节名、原文引用可以保留引号，普通说明文字不要用引号制造概念感。

## 2. AI 味的判断标准

AI 味通常不是由某一个词造成的，而是由句式、词汇、段落、语气和信息密度共同造成的。审阅时先判断问题类型，再决定改写的范围。

| 问题类型 | 常见表现 | 处理方式 |
|---|---|---|
| 意义包装过度 | 把普通事实写成重大意义、历史转折、深刻启示 | 回到具体事实、具体作用和实际结果 |
| 结构过于工整 | 每段都首先、其次、最后，每句话长度接近，结尾都升华 | 按内容关系组织句子，不强行三点式 |
| 表达过于圆满 | 每个概念都解释完整，每个判断都平衡收束 | 保留必要判断，删掉无信息量补充 |
| 词汇过于泛化 | 关键、赋能、助力、格局、复杂性、深度融合等词堆叠 | 写清具体对象、具体动作和具体影响 |
| 语气不贴场景 | 聊天像心理咨询，报告像宣传稿，技术说明像广告 | 按场景调整正式度和信息密度 |
| 格式痕迹明显 | 滥用粗体、冒号、破折号、引号、项目符号和口号式标签 | 保留必要格式，删除装饰性格式 |
| 归因含糊 | 专家认为、多个来源显示、相关研究表明但没有来源 | 有来源就写清来源，没有来源就改为直接判断或删除 |
| 助手话术残留 | 当然、好问题、希望这对你有帮助、如果你愿意 | 删除对话套话，直接进入正文 |
| 模型边界残留 | 截至某日期、根据我的训练数据、基于可用信息 | 需要核验时直接提示核验，不保留模型身份话术 |
| 刻意显得全面 | 强行列出三层、三方面、全方位、多维度 | 内容需要几项就写几项，不凑结构 |

判断时不要只做同义词替换。同义词替换只能解决表层问题。

## 3. 工作流程

### 3.1 判断文本场景

先判断文本属于哪一类：

- 学术论文、课程论文、实验报告
- 技术说明、工具介绍、项目文档
- 心得体会、总结、申请材料、思想汇报
- 社交聊天、评论、私信回复
- 观点文章、博客、公众号
- 百科式条目、资料整理、说明性文本
- 英文翻译或中英混合文本

同一段话在不同场景中的改写方式不同。课程论文不能改得太随便，聊天回复也不能写的过于正式，技术说明不能写成宣传文案。

### 3.2 锁定不可改动的信息

改写前先确认并保留以下内容：

- 事实、数据、时间、人物、机构
- 专业术语、公式、实验条件
- 用户明确表达的态度和立场
- 原文基本逻辑顺序
- 正式文本中必须保留的规范表述
- 用户已经给出的限制条件和目标读者

原文信息不足时，不要补写无法确认的内容。可以把空泛句压缩成稳妥一点的表达，但不能把自己的推测写成事实，遇到不确定的事情一定要问，一定要核实。

### 3.3 分层处理文本

按以下顺序处理，切忌不要一开始就大幅重写：

1. **语气层**：判断文本是否像对应场景中的正常表达。
2. **信息层**：删掉没有实际信息的意义升华、重复解释和空泛判断。
3. **句式层**：处理否定式排比、机械设问、戏剧化定性和过度完整的句子。
4. **词汇层**：替换包装词、填充短语、模糊归因和不合语境的比喻词。
5. **段落层**：合并过碎段落，减少提纲式衔接，避免强行三段式。
6. **标点格式层**：减少无意义冒号、引号、破折号、粗体、括号和装饰性符号。

### 3.4 交付前复查

交付前检查：

1. 有没有改变原意、事实、态度或逻辑顺序？
2. 有没有新增原文没有的经历、数据、来源或结论？
3. 有没有把正式文本改得太口语，把聊天回复改得太像客服？
4. 有没有把简单事实包装成宏大意义？
5. 有没有连续三句结构和长度都相似？
6. 有没有使用没有来源的“专家认为”“研究表明”？
7. 有没有为了简洁删掉必要过渡，导致前后句不连贯？
8. 有没有从其他语境借来比喻词、拟人化动词或抽象包装词？
9. 朗读一遍后，是否像新闻稿、宣传稿、客服话术或模型生成的通用回答？

### 3.5 子 agent 调度规则

当运行环境支持 Task、Subagent、Agent 或类似的子任务工具时，复杂文本任务必须优先使用子 agent 处理，不要只在主流程中模拟分工。

满足以下任一条件时，启用子 agent：

- 原文较长，超过约 500 字，或包含多个段落。
- 用户同时要求处理自然度、正式度、逻辑、语气、AI 痕迹、降重或润色。
- 用户要求先审阅再改写，或要求说明修改原因。
- 文本包含多个场景，例如报告正文、聊天回复、申请材料混在一起。
- 原文信息密度高，包含数据、术语、公式、时间、人物或机构。

以下情况可以不启用子 agent：

- 原文很短，只需要改一句聊天回复。
- 用户明确要求快速直接改写。
- 当前环境没有可用的子 agent 或任务分发工具。
- 调用子 agent 会破坏用户要求的即时性或简洁输出。

如果子 agent 工具不可用，主流程仍按下面的分工顺序内部执行，但不要声称已经调用了子 agent。

#### 3.5.1 场景判断 agent

负责判断文本用途和改写边界，只输出执行改写所需的信息。

输入内容：

- 用户要求
- 原文
- 必要的场景规则

输出内容：

- 文本场景
- 应保持的正式度
- 必须保留的信息
- 不适合使用的语气
- 用户的特殊限制

不要输出长篇解释，也不要改写正文。

#### 3.5.2 AI 痕迹审阅 agent

负责找出原文中真正影响自然度的问题。

检查重点：

- 模板句、宣传稿语气、客服式表达或心理咨询式表达
- 过度升华、空泛总结、机械排比和无来源判断
- 抽象包装词、嫁接词、拟人化表达和不贴场景的词
- 过度整齐的段落结构和装饰性格式
- 句子过碎、过短，导致前后衔接不足的问题

输出时只列主要问题和对应位置，不要把所有可疑词都机械标出。不要改写正文。

#### 3.5.3 改写执行 agent

负责根据场景判断和审阅结果改写正文。

执行要求：

- 先保留原意，再处理表达。
- 先改最明显的 AI 痕迹，再调整词句细节。
- 不为了简洁把句子切得过碎。
- 不为了自然把正式文本改成聊天语气。
- 不添加原文没有的信息。
- 必要时保留过渡词、虚词和长句，让前后句衔接自然。

长文本可以分段改写，但全文语气要一致。

#### 3.5.4 复查 agent

负责检查改写结果是否偏离用户要求。

复查重点：

- 是否改丢事实、态度、术语、数据或逻辑关系。
- 是否新增原文没有的经历、来源、结论或评价。
- 是否把正式文本改得太随意，或把聊天回复改得太像客服。
- 是否仍有明显模板句、空泛词、宣传语和过度升华。
- 是否出现过度短促、句子衔接不足的问题。
- 是否用了为了去 AI 味而制造的不自然表达。

复查后只提出必要修改，不重新大改全文。

#### 3.5.5 主流程合并

主流程负责统一合并各子 agent 的结果，并决定最终输出形式。

输出规则：

- 用户只要求改写时，直接输出改写结果。
- 用户要求说明修改原因时，只说明主要改动，不展示完整内部审阅过程。
- 用户要求对照时，可以按“原句—问题—改写”展示。
- 用户没有要求评分时，不主动输出评分表。
- 子 agent 的中间判断只服务于改写，不要把内部流程完整暴露给用户。

#### 3.5.6 子 agent 上下文控制

调用子 agent 时，不要把整个技能全文都传给每个子 agent。每个子 agent 只接收当前阶段需要的规则和原文。

推荐方式：

- 场景判断 agent 只接收场景规则、用户要求和原文。
- AI 痕迹审阅 agent 只接收判断标准、慎用句式、慎用词汇和原文。
- 改写执行 agent 只接收场景判断、主要问题、改写原则和原文。
- 复查 agent 只接收用户要求、原文、改写结果和复查清单。

这样可以减少上下文干扰，避免每个子 agent 同时处理全部规则。

## 4. 场景规则

### 4.1 学术、课程和实验报告

这类文本要清楚、克制、准确。改写时保留学术性，不要改成聊天的语气。

保留：

- 专业术语
- 必要定义
- 推导过程
- 数据和实验条件
- 规范章节结构
- 必要的因果、转折和条件说明

减少：

- 具有重要意义
- 提供了新的思路
- 为后续研究奠定基础
- 从多个维度展开
- 深入探讨其复杂性
- 理论与实践相结合的重要体现
- 充分体现了学生的综合素养

写法：

- 先写研究对象，再写方法，再写结果或问题。
- 用具体作用代替宏大评价。
- 能说明实验误差、参数变化、对比结果，就不要写空泛意义。
- 内容需要几项就写几项，不强行写成三点。
- 可以写限制和不足，少用空泛乐观结尾。

示例：

原句：
> 本文深入探讨了该问题的复杂性，并从多个维度展示了其重要意义。

改写：
> 本文分析该问题的形成原因，并比较两种常见处理方法的差异。

原句：
> 本实验不仅帮助学生深入理解理论知识，更培养了学生的实践能力和科学素养，体现了理论与实践相结合的重要意义。

改写：
> 本实验把理论公式和实际测量过程对应起来，也暴露出仪器读数、参数设置和误差处理中的具体问题。

### 4.2 技术说明、工具介绍和项目文档

这类文本优先保证可验证、可复现、可理解。工具介绍不要写成广告。

优先写：

- 发布时间或版本
- 开发者、组织或维护方
- 许可证或开源情况
- 核心架构或基本原理
- 主要功能
- 当前任务中的具体用途
- 使用限制、依赖条件和已知问题

慎用：

- 拥有
- 凭借
- 完美支持
- 丝滑体验
- 无可比拟
- 理想工具
- 强大生态
- 一站式解决方案
- 深度融合
- 赋能开发者

写法：

- 直接说明功能、输入、输出、依赖和使用步骤。
- 能写具体功能，就不写“强大”。
- 能写限制条件，就不写“完美支持”。
- 技术文档可以有列表，但列表要服务操作，不要用于装饰结构。

示例：

原句：
> 该工具凭借强大的功能和丝滑的体验，成为数据分析的理想工具。

改写：
> 该工具支持数据清洗、统计分析和图表绘制。在本任务中，它主要用于整理实验数据并生成折线图。

### 4.3 心得体会、总结、申请材料和思想汇报

这类文本要具体、可信，并写出后续行动。不要只写表态和口号。

常见问题：

- 通过本次学习，我深刻认识到……
- 在今后的学习生活中，我将……
- 作为新时代青年……
- 不断提升自身综合素质
- 坚定理想信念，勇担时代使命
- 以更高标准要求自己

写法：

- 少写“我认识到”，多写“我之前怎么做、问题在哪里、之后怎么改”。
- 正式材料可以保留必要术语，但不要反复堆同类表态。
- 结尾写具体行动，不要空泛升华。
- 个人材料要有真实经历支撑，不能替用户编造经历。

示例：

原句：
> 通过本次学习，我深刻认识到理论联系实际的重要性。在今后的学习生活中，我将不断提升自身综合素质。

改写：
> 这次学习让我发现，自己之前更重视结论，忽略了推导过程。后面复习时，我会先把基本概念说清楚，再做题验证。

### 4.4 聊天回复和社交表达

聊天回复要像正常交流，避免客服、心理咨询师或公众号作者的腔调。

慎用：

- 我能感受到你……
- 听起来你现在……
- 这确实是一个值得……的问题
- 你的感受是完全合理的
- 如果你愿意的话，可以……
- 过度总结对方情绪
- 把简单回应写成情感分析

写法：

- 直接回应对方说的具体内容。
- 情绪表达短一点，不分析太满。
- 根据双方关系决定口语程度，不能一律加“啊、呀、呢、吧”。
- 用户没有要求特别口语化时，避免刻意松散。
- 不要使用没有真实语境支撑的抽象口语。

聊天回复里尤其慎用这些词：

- 圆上了
- 接住了
- 那种感觉
- 情绪价值
- 共鸣感
- 破防了
- 宿命感

这些词在某些真实语境中可以用，但不能当作通用自然化工具。更稳妥的做法是回应事件本身和对方情绪。

反例：
> 就是等了很久，终于圆上了那种感觉。

改写：
> 等了这么久，总算有结果了，肯定很激动。

更口语的版本：
> 终于等到了，太不容易了。

### 4.5 观点文章、博客和公众号

观点可以直接写，语气保持平实。不要把普通观点包装成宏大的命题。

慎用：

- 在当今社会……
- 这给我们带来了深刻启示……
- 时代浪潮下……
- 每段都升华
- 每个概念都加引号
- 把普通观点包装成宏大命题

写法：

- 用具体问题开头。
- 观点直接说，不绕成背景介绍。
- 可以保留有限度的不确定。
- 少用整齐排比，多用自然推进。
- 结尾可以收束观点，但不要写成宣传口号。

示例：

原句：
> 这一现象反映了当代社会结构变化下个体选择的复杂性。

改写：
> 这个现象背后有社会结构的影响，但个人选择也不能完全被忽略。

### 4.6 百科式、资料整理和说明性文本

这类文本要中立、具体、可查。对象的地位、影响和象征意义只在有事实支撑时才写。

慎用：

- 重要角色
- 持久影响
- 广泛认可
- 充满活力
- 自然美景
- 文化遗产
- 著名的
- 迷人的
- 必游之地
- 坐落于……中心

写法：

- 直接写事实、职责、时间、地点、功能、作品或事件。
- 有来源时写清来源内容。
- 没有来源时不写模糊结论。
- 少用旅游推广、品牌宣传和荣誉总结式语言。

示例：

原句：
> 该机构的成立标志着区域治理发展史上的关键时刻，并为当地行政现代化做出了重要贡献。

改写：
> 该机构成立于 1989 年，负责收集和发布区域统计数据。

### 4.7 英文翻译和翻译腔中文

翻译时要让中文连贯通顺，避免逐词对应。

常见问题：

- 值得注意的是
- 在……的背景下
- 当涉及到……时
- 发挥着关键作用
- 一个……的例子是
- 这突显了……的重要性

写法：

- 先理解英文意思，再按中文习惯重组句子。
- 能用主动句就少用被动句。
- 能直接说原因、结果、条件，就不要保留英文式结构。
- 保留术语准确性，不为了自然牺牲专业含义。

示例：

原句：
> 当涉及到学习效率时，时间管理发挥着关键作用。

改写：
> 学习效率很大程度上取决于时间管理。

## 5. 句式规则

### 5.1 优先直接陈述

能直接说清楚时，不要先排除反面再绕回正面。过多否定式排比会让文本显得很机械。

常见问题：

- 不能……也不能……
- 不是……也不是……
- 不应……而应……
- 不要把……当成……

改法：直接写应该怎么做。

反例：
> 去 AI 味不能把文本一律改成口语，也不能靠错别字、病句或松散表达来制造自然感。

改写：
> 去 AI 味要保留文本原本的场景、事实和语气，再处理模板化表达。自然感来自准确、具体、顺畅的中文。

### 5.2 保留必要连贯性

自然写作需要节奏，也需要前后句衔接。删去空话不代表把句子切碎，文字少也不一定好读，过度精炼会让文本显得生硬。

可以保留这些轻量连接：

- 因此
- 所以
- 也
- 再
- 其中
- 这种
- 这类
- 此时
- 这样
- 更准确地说
- 从这个角度看

这些词本身不是 AI 味。只要它们确实承担逻辑关系，就可以保留。

反例：
> AI 味通常由几类问题叠加形成。单独替换某个词往往不够。

改写：
> AI 味通常由几类问题叠加形成，因此单独替换某个词往往不够。

反例：
> 句子太整齐。段落太完整。结尾太像总结。

改写：
> 如果句子过于整齐，段落又总是完整收束，文本就容易显得像模板生成。

### 5.3 减少否定式排比

尽量少用：

- 不是……而是……
- 不仅仅是……更是……
- 这不是简单的……而是……
- 这不仅是一种……也是一种……

改法：直接陈述判断，不用修辞铺垫。

原句：
> 这不仅是一种技术改进，更是工作方式的深刻变化。

改写：
> 这项技术改进改变了部分工作流程。

原句：
> 这不是一次普通的更新，而是产品体验的一次跃升。

改写：
> 这次更新主要改善了产品体验。

### 5.4 减少戏剧化定性

慎用：

- 标志着
- 见证了
- 充当了
- 作为……的证明
- 体现了……的重要意义
- 彰显了……
- 代表着一次重大转变
- 为……奠定坚实基础
- 塑造了……
- 留下了不可磨灭的印记
- 深深植根于……

改法：优先使用“是、有、带来、用于、使、导致、影响、改善、降低、提高”等平实动词。

原句：
> 该系统的上线标志着企业数字化转型迈入新阶段。

改写：
> 该系统上线后，企业可以统一处理订单、库存和客户数据。

原句：
> 这一成果是团队长期创新能力的有力证明。

改写：
> 这一成果来自团队前期的模型设计、实验验证和多轮修改。

### 5.5 减少无结论设问

设问可以使用，但后面要给出明确回答。不要用设问制造思考感。

反例：
> 这是否意味着传统方法已经失效？这个问题值得我们继续思考。

改写：
> 这并不意味着传统方法失效，但说明它在实时处理场景下存在限制。

### 5.6 减少表演式范围句

慎用没有真实尺度关系的“从 X 到 Y”。

反例：
> 本文从理论基础到现实应用，从个体经验到社会结构，全面呈现了该问题的多重面向。

改写：
> 本文先说明理论基础，再结合现实案例分析该问题。

## 6. 词汇规则

### 6.1 高频包装词

以下词不是绝对禁用，但要看是否真的提供信息。若只是让句子显得高级，应删除或改成具体内容。

慎用：

- 此外
- 至关重要
- 深入探讨
- 强调
- 持久
- 增强
- 培养
- 获得
- 突出
- 相互作用
- 复杂性
- 关键
- 格局
- 关键性
- 展示
- 织锦
- 证明
- 宝贵
- 充满活力
- 赋能
- 助力
- 多维度
- 全方位
- 深层次
- 有效提升
- 显著改善
- 自然之美
- 著名的
- 令人叹为观止的
- 迷人的
- 坐落于
- 位于……的中心
- 开创性的
- 必游之地

常见处理：

- 至关重要 → 重要、直接影响……
- 深入探讨 → 分析、说明、比较
- 强调了…… → 说明了……
- 增强了…… → 提高了……、改善了……
- 复杂性 → 写清复杂在哪里
- 关键作用 → 写清起什么作用
- 赋能 → 帮助、支持、使……能够
- 助力 → 帮助、推动、用于
- 多维度 → 写出具体维度

示例：

原句：
> 该方法从多个维度有效提升了系统性能。

改写：
> 该方法降低了计算耗时，也减少了结果波动。

### 6.2 填充短语

少用：

- 值得注意的是
- 为了实现这一目标
- 在……的情况下
- 具有……的能力
- 可以被认为
- 从某种意义上说
- 不难发现
- 由此可见
- 总而言之
- 综上所述
- 在这个时间点
- 由于……的事实
- 在您需要帮助的情况下

改法：能删就删；需要保留逻辑关系时，改成直接表达。

原句：
> 值得注意的是，该算法具有处理大规模数据的能力。

改写：
> 该算法可以处理大规模数据。

原句：
> 在样本数量较少的情况下，模型可能会产生一定误差。

改写：
> 样本数量较少时，模型误差会增大。

### 6.3 模糊归因

避免：

- 行业专家认为
- 观察者指出
- 多个来源显示
- 相关研究表明
- 有观点认为
- 业内人士表示
- 批评者认为
- 许多出版物指出
- 媒体普遍报道

处理方式：

- 有具体来源时，写清来源和内容。
- 没有来源时，改成作者自己的判断。
- 无法确认时，删除或提示需要核验。

原句：
> 行业专家认为，该技术将改变未来教育模式。

改写：
> 这项技术目前主要用于作业批改、学习反馈和课堂辅助。

原句：
> 多个来源显示，该产品受到用户欢迎。

改写：
> 根据平台公开评分，该产品近三个月的平均评分为 4.6 分。

### 6.4 夸张限定和回避表达

避免叠加不确定词：

- 可能潜在地
- 可以被认为
- 可能会产生一些影响
- 在一定程度上可能
- 似乎可以说
- 某种程度上体现出

改法：保留必要不确定性，删除重复限定。

原句：
> 该政策可能潜在地会对结果产生一些影响。

改写：
> 该政策可能会影响结果。

### 6.5 系统化标签和抽象包装词

慎用：

- 管线
- 逻辑链条
- 教学链路
- 双向联动
- 整合为……系统
- 教学共同体
- 三位一体
- 闭环
- 有望
- 建构
- 生态
- 矩阵
- 抓手
- 载体
- 路径

常见处理：

- 有望 → 可、可能
- 建构 → 形成、呈现、建立
- 管线 → 流程、步骤
- 逻辑链条 → 逻辑关系、推导顺序
- 双向联动 → 相互配合、形成关联
- 整合为……系统 → 放在一起使用、统一在……下
- 闭环 → 写清从什么到什么，再如何反馈

原句：
> 本文建构了学习、实践和反馈三位一体的教学闭环。

改写：
> 本文把学习内容、实践任务和反馈修改放在同一流程中。

### 6.6 比喻化、拟人化和嫁接词

有些词本身没有错，但放到不合适的语境里会显得抽象。写作规范、技术说明和课程文本中，优先使用常用词和直接动词。

慎用：

- 接得住
- 圆上了
- 那种感觉
- 喂给
- 跑起来
- 打通
- 承载
- 沉淀
- 抓手
- 牵引
- 赋予生命
- 让数据说话
- 让文本呼吸
- 与读者对话
- 让系统具备感知能力

常见处理：

- “让句子之间接得住” → “让前后句衔接自然”
- “终于圆上了那种感觉” → “总算有结果了”
- “把数据喂给模型” → “把数据输入模型”
- “流程跑起来” → “流程可以执行”
- “打通前后环节” → “连接前后环节”
- “沉淀为方法” → “整理成方法”
- “作为重要抓手” → “作为主要措施”
- “让文本呼吸” → “让句子节奏更自然”

改写时优先选择普通词。能用输入、执行、连接、整理、形成、呈现、说明、影响这些词说清楚，就少用带比喻感和拟人感的表达。

## 7. 段落和格式规则

### 7.1 少用程式化衔接词

少用：

- 首先
- 其次
- 再次
- 最后
- 一方面
- 另一方面
- 与此同时
- 此外
- 然而
- 综上所述

这些词不是绝对不能用。问题在于用它们凑结构，而不是表达真实的层次关系。

原句：
> 首先，本文介绍了研究背景。其次，本文分析了相关理论。最后，本文提出了改进建议。

改写：
> 本文先介绍研究背景，再分析相关理论，并在此基础上提出改进建议。

### 7.2 合并过碎段落

AI 文本常把一句话拆成一段，造成说明书感。若几个短段共同说明同一问题，应合并。

原文：
> 本方法操作简单。  
> 结果较为稳定。  
> 适合初学者使用。

改写：
> 本方法操作简单，结果较稳定，适合初学者使用。

### 7.3 避免强行三段式完整

内容需要几项就写几项。两点、四点或直接成段都可以。

原句：
> 该活动提升了能力、开阔了视野、增强了信心。

改写：
> 这次活动让我更熟悉实际流程，也暴露出自己在时间安排上的问题。

### 7.4 调整句子节奏

检查是否连续出现相同长度、相同结构的句子。连续三句相似时，至少改动一句。

处理方法：

- 用一个短句承接判断。
- 用一个较长的句子解释原因或条件。
- 用自然过渡句连接上下文。
- 不要每句话都写成“主语 + 通过…… + 实现了……”。

### 7.5 减少解释型冒号

避免总起词加冒号再展开说明的说明书式句子。能自然接上就自然接上。

原句：
> 标准做法是：先收集数据，再清洗数据，最后建立模型。

改写：
> 标准做法通常是先收集数据，再清洗数据，最后建立模型。

原句：
> 主要问题是：样本数量不足，评价指标单一。

改写：
> 主要问题在于样本数量不足，评价指标也比较单一。

### 7.6 删除无实际作用的引号

中文引号只在必要时使用。正式术语、作品名称、栏目名称、文件名、章节名、引用原话、反例句式可以保留。普通说明文字不要用引号包装短语，不要用引号制造概念感。

反例：
> 本文尝试构建一种“可理解、可操作、可验证”的学习路径。

改写：
> 本文尝试把学习过程拆成理解概念、完成操作和验证结果三个环节。

反例：
> 本方法可以让文本从“机器感”走向“真实表达”。

改写：
> 本方法可以减少文本中的机器感，让表达更接近实际写作。

### 7.7 减少长破折号串联

长破折号常被用来制造节奏感，但过多时会像文案。概念之间可以用普通句子连接。

原句：
> 本文按照问题提出—理论分析—案例验证—结论总结的路径展开。

改写：
> 本文先提出问题，再分析相关理论，并结合案例验证结论。

### 7.8 并列内容优先用顿号

中文并列名词优先使用顿号，少用英文逗号或括号堆叠。

原句：
> 该模型考虑了成本，收益，风险和效率。

改写：
> 该模型考虑了成本、收益、风险和效率。

### 7.9 结果表达不额外加括号

原句：
> 计算得到均衡价格为（20元）。

改写：
> 计算得到均衡价格为20元。

### 7.10 删除机械逗号

以下表达中，衔接词后通常不需要逗号：

- 然后我们得出结论
- 因此我们认为
- 由此可以看出
- 接下来分析
- 下面讨论

原句：
> 因此，我们认为该方法具有一定优势。

改写：
> 因此我们认为该方法有一定优势。

### 7.11 避免装饰性格式

用户明确要求保留格式时再保留；其他情况减少粗体、表情符号和标题加说明式列表。

原句：
> 🚀 **启动阶段：** 产品在第三季度发布。

改写：
> 产品在第三季度发布。

## 8. 常见 AI 写作模式

### 8.1 过度强调意义、遗产和趋势

表现：

- 把普通事件写成关键转折点。
- 把具体事实拔高成更广泛趋势。
- 反复使用标志着、反映了、象征着、体现了、贡献于。

处理方式：直接写发生了什么、谁做了什么、结果是什么。

原句：
> 该机构的成立标志着区域治理发展史上的关键时刻。

改写：
> 该机构成立后，负责收集和发布区域数据。

### 8.2 过度强调知名度和媒体报道

表现：

- 反复列举媒体名称。
- 只说受到广泛关注，不说关注点是什么。
- 用粉丝数、报道数制造影响力。

处理方式：写具体报道内容、具体评价或具体事件。

原句：
> 她的观点被多家国内外媒体引用，并在社交平台上拥有活跃影响力。

改写：
> 在一次采访中，她主张监管应关注 AI 系统造成的结果，不只审查技术方法。

### 8.3 句末追加肤浅分析

表现：句子已经完整，末尾又追加“从而、确保、反映、象征、彰显”等短语，制造虚假深度。

处理方式：删除尾部泛化判断，或改成具体结果。

原句：
> 该设计采用蓝色和绿色，象征着自然、开放和可持续发展理念。

改写：
> 该设计采用蓝色和绿色。设计说明中称，这两种颜色对应水域和植被。

### 8.4 宣传和广告式语言

表现：

- 使用充满活力、丰富、深刻、迷人、令人叹为观止等形容词。
- 把普通地名、产品、活动写成推广文案。
- 用坐落于、拥有、展示、致力于等词包装对象。

处理方式：改成中立事实。

原句：
> 这座充满活力的城市拥有丰富的文化遗产和迷人的自然景观。

改写：
> 这座城市有两处历史建筑和一个沿河公园。

### 8.5 提纲式挑战与未来展望

表现：

- 先写“尽管面临挑战”，再写“仍然继续发展”。
- 结尾写“未来可期”，但没有具体内容。
- 套用挑战、机遇、展望的固定结构。

处理方式：写具体问题和具体措施。

原句：
> 尽管面临交通拥堵和资源不足等挑战，该地区仍将继续蓬勃发展。

改写：
> 新园区建成后，通勤拥堵加重。当地在 2022 年启动了道路扩建和公交线路调整。

### 8.6 回避简单系动词

表现：用“作为、代表、充当、拥有、设有”替代简单的“是、有”。

处理方式：优先使用“是”和“有”。

原句：
> 该中心作为学校的综合服务平台，设有多个功能区域。

改写：
> 该中心是学校的综合服务平台，有咨询、预约和资料查询三个区域。

### 8.7 刻意换词

表现：为了避免重复，连续使用同义词，导致指代混乱。

处理方式：需要重复时就重复，清楚优先。

原句：
> 主人公面临许多挑战。主要角色必须克服障碍。中心人物最终获得胜利。

改写：
> 主人公面临许多挑战，但最终克服困难并获得胜利。

### 8.8 协作交流痕迹

避免：

- 当然！
- 好问题！
- 你说得完全正确！
- 希望这对你有帮助
- 如果你还想了解，请告诉我
- 下面是……
- 我来帮你……

处理方式：删除对话痕迹，直接进入正文。

原句：
> 当然！下面是关于法国大革命的概述，希望这对你有帮助。

改写：
> 法国大革命始于 1789 年，财政危机和粮食短缺是重要背景。

### 8.9 知识边界免责声明

避免：

- 截至……
- 根据我最后的训练更新
- 基于可用信息
- 虽然具体细节有限
- 我无法访问实时信息

处理方式：如果有事实来源，直接写事实；如果没有，改为稳妥表达或提示需要核验。

原句：
> 基于可用信息，该公司似乎成立于 20 世纪 90 年代。

改写：
> 现有资料显示，该公司成立于 20 世纪 90 年代。具体年份仍需核验。

### 8.10 谄媚和过度积极

表现：

- 过度称赞用户问题。
- 反复确认“你完全正确”。
- 不必要地显得顺从和热情。

处理方式：只回应实际内容。

原句：
> 好问题！你说得完全正确，这是一个非常复杂且重要的话题。

改写：
> 你提到的因素在这里确实相关。

## 9. 质量评分

用户没有要求时，不必每次输出评分表。需要自查时，可按 5 个维度判断。

| 维度 | 评估标准 |
|---|---|
| 直接性 | 是否直接陈述事实，少绕圈宣告 |
| 节奏 | 句子长度和结构是否有变化，是否没有过度切碎 |
| 场景匹配 | 正式度、口语程度和信息密度是否符合使用场景 |
| 事实稳定 | 是否保留原文事实、术语、数据和限制条件 |
| 精炼度 | 是否删掉无信息量内容，同时保留必要过渡 |

判断标准：

- 45-50 分：基本去除 AI 痕迹
- 35-44 分：仍有少量模板感
- 低于 35 分：需要继续修订

评分只用于辅助判断，不能代替实际改写。若评分结果和朗读感受冲突，以文本实际效果为准。

## 10. 示例库

### 示例 1：软件更新

原句：
> 新的软件更新作为公司致力于创新的证明。此外，它提供了无缝、直观和强大的用户体验，确保用户能够高效地完成目标。这不仅仅是一次更新，而是我们思考生产力方式的革命。行业专家认为这将对整个行业产生持久影响，彰显了公司在不断演变的技术格局中的关键作用。

改写：
> 这次软件更新加入了批处理、键盘快捷键和离线模式。测试用户的反馈较好，多数人表示任务完成速度比之前更快。

修改点：

- 删除“作为……的证明”。
- 删除“此外、无缝、直观、强大”等包装词。
- 删除“这不仅仅是……而是……”。
- 删除“行业专家认为”等模糊归因。
- 用具体功能和反馈代替宏大判断。

### 示例 2：课程报告

原句：
> 本实验不仅帮助学生深入理解理论知识，更培养了学生的实践能力和科学素养，体现了理论与实践相结合的重要意义。

改写：
> 本实验把理论公式和实际测量过程对应起来，也暴露出仪器读数、参数设置和误差处理中的具体问题。

### 示例 3：技术工具介绍

原句：
> 该平台凭借强大的生态和灵活的扩展能力，为开发者提供了一站式解决方案。

改写：
> 该平台提供模型调用、数据管理和接口调试功能。开发者可以在同一环境中完成测试和部署前检查。

### 示例 4：社交回复

原句：
> 我能感受到你现在真的很开心，这种等待很久终于实现目标的感觉一定很珍贵。

改写：
> 等了这么久，总算有结果了，肯定很激动。

更口语的版本：
> 终于等到了，太不容易了。

### 示例 5：心得体会

原句：
> 通过本次活动，我深刻认识到团队合作的重要性，也更加坚定了以后不断提升自身综合素质的决心。

改写：
> 这次活动让我发现，自己在分工沟通上还不够主动。后面再做小组任务时，我会提前确认每个人的进度，避免问题拖到最后才处理。

## 11. 最终原则

1. 先判断场景，再决定改写力度。
2. 保留事实、术语、数据、态度和逻辑顺序。
3. 不额外添加原文没有的经历、来源和结论。
4. 优先具体表达，减少抽象判断。
5. 优先直接表达，减少包装和修辞铺垫。
6. 能用“是、有、做了什么”说清楚，就少用戏剧化定性。
7. 能删掉的空话就删掉，但不要删掉必要过渡。
8. 正式文本保留正式度，聊天回复保留交流感。
9. 不把自然感等同于短句、口语、错别字或松散表达。
10. 少用从其他语境借来的比喻词和拟人化说法。
11. 如果一句话朗读起来像新闻稿、宣传稿、客服话术或模型生成的通用回答，就改得更具体、更直接。
